CSS盒模型（box model）

CSS盒子模型就是指对于页面上的每一个元素（无论是块级元素还是行内元素）都被看做一个矩形框，由元素内容、内边距、边框、外边距组成。盒子决定了该元素在页面中的占位。值得一提的是标准盒子模型与混杂盒子模型是有差别的。
标准盒子模型的实际占位：width+padding*2+border*2+margin*2
混杂盒子模型的实际占位：width+margin*2

如何解决混杂盒子模型带来的不便：
我们可以避免混杂模式的触发。通过文档类型声明DOCTYPE，可以选择是触发混杂模式还是标准模式。通过设置文档类型声明，可以告诉浏览器使用哪种文档类型定义DTD，以及选择哪种HTML和网页呈现的方式。当页面以混杂模式呈现时，其行为是不可预知的，所以要尽量避免混杂模式的触发。混杂模式通常是为了模拟老版的浏览器使其仍能够正常运行。

注：在CSS2中背景应用于内容和内边距组成部分；而在CSS2.1中背景已经延伸到了边框（这也是CSS标准规定的）

在CSS3中这个是可设的。

内边距

内边距的值可以是百分比，是相对与其父元素的宽度width来计算的，不管是垂直内边距或水平内边距都是根据父元素的width计算的。

边框

需要注意的是，如果你希望有边框就一定要记得设置样式border-style，因为默认是none，即便设置了边框宽度和颜色也不会显示的。

若没有设置边框颜色，那么边框颜色默认为元素文本的颜色，若当前元素无文本那么边框颜色就是其父元素的文本颜色。

外边距

和内边距一样，给外边距设置值为百分比时，是相对于其父元素的宽度width来计算的。
对于行内元素来讲，垂直的外边距不会产生效果。

写在前面

ajax就是使用异步的方式从服务器获取数据或者向服务器写入数据的技术。它使用户无需刷新整个页面就能从服务器获取数据，并且在发送请求后用户不用傻等还能继续其他操作。

之前一直用jQuery封装AJAX的各种接口，用的特顺手，后来为了系统学习特意买了本《Head First Ajax》，看了大概5章实在有些受不了了，这本书节奏太慢，感觉是针对完全的js小白。于是，转看《js高级程序设计》了，这本书讲的Ajax相关知识算是比较广的。这部分在练习的时候需要和后台结合才能较深理解Ajax工作机制，在练习过程中遇到了一些问题：后台无法获取参数数据、XHR全局变量问题、readystatechange事件中event对象的问题。详细感悟如下。

XHR对象

XMLHttpRequest对象是ajax的核心，js就是通过XHR对象向服务器发送异步请求并处理响应的。
XHR对象的主要属性和事件

  • responseText
  • responseXML
  • readystate
    • 0（未初始化），XHR对象已建立，但未初始化（即未调用open方法）
    • 1（初始化），已初始化，但还未发送数据（即未调用send方法）
    • 2（发送数据），已发送数据，响应和http头未知
    • 3（数据接收中），已接收部分数据，响应和http头不全
    • 4（完成），数据接收完毕。
  • status
    • 1xx 信息提示
    • 2xx 成功
    • 3xx 重定向
    • 4xx 客户端错误
    • 5xx 服务器错误

创建XHR对象（跨浏览器兼容）

get与post的比较

  • 用途
    GET通常向服务器请求查询数据；无论GET多少次都不会改变服务器资源；
    POST通常向服务器发送需要处理的数据；会改变服务器资源。

  • 参数数据发送过程
    GET通过open()向服务器发送查询字符串，查询字符串就是url的一部分，且一般需要进行encodeURIComponent编码;
    POST通过send()向服务器发送参数，参数跟着http消息发送。

  • 服务器获取参数数据的方式
    使用GET时，由于参数时url的一部分所以只能是文本类型，服务器默认按照文本数据的方式解析；
    在使用POST时，不只能发送纯文本，还能发送很多其他类型的数据，因此必须明确告诉服务器发送的是什么类型数据，这样服务器才能正确解码。需要设置XHR对象的请求头部信息”Content-Type”为”application/x-www-form-urlencoded”
    测试了下，在C#下GET时服务器通过Request.Params["name"]才能正确获取参数name的值；POST时服务器通过Request.Form["name"]才能正确获取参数name的值。

  • 数据量
    GET 传送数据量较小，有长度限制。这是由于GET是将数据添加到URL中，而浏览器将URL的最大长度限制在2k；
    POST 传送数据量是不受限的。

  • 安全性
    POST要比GET安全，这是 由于： 

    - GET的数据都是添加进URL,对所有人可见；POST的传送的数据是不可见的；  
- GET的数据能够被缓存，记录在历史记录里；POST的数据不能被缓存，不会留在浏览记录里。

  • 效率
    POST比GET更加耗资源，效率较低，POST发送数据是分open()和send()两步进行的。

ajax的原理和机制

从上述可以总结出使用AJAX发送一次请求的步骤：

  • 首先创建XHR对象（要考虑到浏览器的兼容性）；
  • 给该XHR对象绑定readystatechange事件；
  • 调用open方法，选择使用get还是post方式发送请求；
  • 若选择get方式则将查询字符串写入url中（注意，参数需要编码）；
  • 若选择post方式则使用send方法发送参数；
  • 在readystatechange事件处理程序中，判断请求对象的readystate和status属性，以便响应成功后进行处理。
  • 收到服务器的响应后，响应的数据会自动填充XHR对象的属性：
    • responseText,存放返回的文本，无论响应的内容是什么类型其内容都会保存在这个属性中
    • responseXML，如果响应的内容类型是”text/xml”或”application/xml”，这个属性存放返回的包含响应数据的XML DOM文档；如果不是xml类型，该属性为null
    • status，响应的http状态
    • statusText,响应的http状态说明
  • js可以根据上述值做出处理。

    • 1.你们公司前端团队大概多少人；你们前端和后台的合作方式。（一方面这个问题的确是我最关心的；另外一方面你向面试官传达了你对他们公司真心感兴趣）
    • 2.你们公司前端都做什么产品（你可以通过这个衡量你是否愿意进这个公司做这个产品，很多面试官特别乐意介绍）
    • 3.根据对我1个小时的了解，能不能给我提写建议，以此提高自己（可以看出他对你的态度）

  • 字符串之间：这个比较简单，逐个字符进行比较ASCII码值。

  • 字符串与数值之间：将字符串转换数值，然后比较数值大小。 

    alert(10 < "11c");//false
alert(10 > "11c");//false

    上述代码两次均返回false，这是由于"11c"不能转化为数值。可以测试isNaN("11c")返回true。
    数值与NaN比较大小永远返回false。
    需要注意的是，此处字符串转换为数值的规则与parseInt()是不相同的。此处的规则与Number()和isNaN()是一致的，规则如下:

    • 若字符串只包含数字、正号+ 和负号-，转化为十进制。例如： 

      alert(+"10" > 9);//true 即(10<9) 
 alert(+"10>"9");//true 即(10>9)
   alert(-"10"<9);//true 即(-10<9)
  alert("-10"<9);//true 即(-10<9)
    • 若包含有效的浮点格式，例如2.1，则转化为响应的浮点数
    • 若为空字符串，则转换为0

    • 若包含0x则转换为十六进制，若包含0则不按照八进制解析而忽略头部的任意多个0； 

      alert("011"<9)//false 即(11<9)  
alert("0x11"<15)//false 即(17<15)

    • 若包含以上格式其他字符，则返回NaN，例如 

      alert(10 < "11c");//false 即(10<NaN)
   alert(10 > "11c");//false 即(10>NaN)

parseInt()工作原理

parseInt(string,radix)用法：

  • 返回值是十进制；
  • 无视string中最前和最后的空格；
  • 参数radix是指解析string的进制，若未指定radix或值为0则按默认进制对string进行解析；

  • 未指定radix的情况下，若string的首字符为0，则按照八进制进行解析；若string的首字符为0x则默认按照十六进制解析string，其他情况默认为十进制； 

    parseInt("011");//9
parseInt("0x11");//17

  • 解析从第一个字符开始，到第一个不能解析的字符（空格、英文、标点等）结束；重点说一下，能解析的字符包括数字、正号+和负号-； 

    parseInt("1a");//1
parseInt("");//NaN
parseInt("-1");//-1

  • 若第一个字符就不能解析，则返回NaN；

实例一： 

var c = parseInt("017/08/2009");
alert(c);               //15

解析：未指定radix，第一个字符为0，则按照八进制进行解析，且解析到7停止，017输出15。

实例二： 

var d = parseInt("09/08/2009");
alert(d);              //0

解析：未指定radix，第一个字符为0，按照八进制解析时9属于不能解析的字符，故0输出0.

Number()和parseInt()区别（实例）

Number("1a");//null
parseInt("1a");//1

Number("");//0
parseInt("");//NaN

Number("011");//11
parseInt("011");//9

类数组对象：

类数组对象归根结底是一个对象，它是Object的一个实例，而不是Array的实例；
由上可知，类数组对象不能调用大多数组的方法（join、slice、concat等）；
